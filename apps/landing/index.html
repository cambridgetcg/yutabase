Not a database engine. A standard installed on top of boring Postgres: ~200 lines of SQL, a 500-line TypeScript client, and a language of six verbs that compiles to SQL you can read. Rollback is DROP SCHEMA.
see it work read the creed

01three sentences of creed

straightforward
every primitive is a one-word rename of something Postgres already does well; nothing is hidden, everything compiles to SQL you can read
organised
one place for vocabulary, one place for connections, one shape for every record's provenance
connected by words
a relation without a word does not exist. submissions/X contains items/Y is data. An anonymous foreign key is not a sentence, so here it is not a relation.

03YOUSPEAK — six verbs, frozen

The dialect fits in a memory file. Six verbs. Every proposed seventh is a request to write SQL instead.

Traversal caps at 2 hops. Deeper means you write WITH RECURSIVE yourself, knowingly.

.how / .at / .by / .src address the honesty header in any where.

04the honesty header

Every deck and every thread carries four columns. There are no SQL defaults for how and by — a write that doesn't say is refused.

at
timestamptz — when this became true
by
who asserted it: human:yu, agent:claude/session, system:cron/job
how
CHECK-constrained: one of the five claim kinds
src
text[] — what it came from; required for cached and computed

The five claim kinds:

witnessed — a human saw or did it
live — read from the authoritative source at at
cached — a copy that may be stale; src says of what
computed — derived; src lists the input refs
declared — asserted without evidence

The honest ceiling, stated: provenance is self-reported. An agent can stamp witnessed falsely. YUTABASE makes lying explicit and auditable — not impossible.

05the starter lexicon — seven words

A word is born by migration. No gloss, no word. No inverse reading, no word. Banned by name: related_to, linked, refs, misc.

Seven words. Five spare in the budget. That's the point.

06annex, don't rewrite

A book is annexed one domain at a time into the existing database. Zero data movement. Rollback = drop two schemas.

install
$ yuta init --conn postgresql://localhost/mydb applying 0001_yu_core.sql... done applying 0002_starter_lexicon.sql... done YUTABASE installed. The vocabulary lives with the data. Seven words coined. Five spare in the budget. That's the point.

It touches no existing table. The yu schema lives beside everything. The vocabulary lives with the data, glosses included. If every tool above it dies, the data and its meaning remain readable by any future hand with nothing but psql.

07the refusals

YUTABASE deliberately does not have:

no ORM, no model classes, no query builder — strings in, rows out
no storage engine, no wire protocol, no replication — Postgres's are better
no graph ambitions — 2-hop cap; recursion is hand-written SQL
no anonymous relations — related_to is banned by name
no silent provenance — a write that won't say how/by bounces
no hosted offering, no telemetry, no recurring spend
